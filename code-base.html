一、项目概述

本项目是面向大学城场景的外卖小程序平台，覆盖用户点餐、商户接单、骑手配送、平台管理四大核心角色，支持微信支付与平台抽成分账机制。小程序端使用微信原生开发（WXML/WXSS/JavaScript），后台管理端为 Web 网页，服务端采用 NestJS。

用户端
浏览商家、下单支付、实时追踪
申请商户 / 骑手入驻
商户端
管理店铺与商品
接单备餐（认证后解锁）
骑手端
接单配送
实时位置上报（认证后解锁）
后台管理
Web 网页（PC 端）
审核 · 佣金配置 · 数据统计

1.1 四端角色说明

端角色主要功能技术形态
用户端普通用户浏览商家、下单支付、申请入驻、实时追踪配送微信小程序（原生）
商户端认证商户管理店铺信息、上架商品（需审核）、查看/处理订单微信小程序（动态 Tab 解锁）

1.2 核心业务流程

微信授权登录→ 申请商户/骑手→ 后台审核→ 解锁对应 Tab
商户上架商品→ 后台审核商品→ 用户下单支付→ 商户接单备餐→ 骑手取餐配送→ 用户确认收货→ 平台触发分账

2.1 整体技术栈

层级技术说明
小程序前端微信原生 WXML/WXSS/JS无框架依赖，直接调用微信全部原生 API，性能最优，调试体验最好

主数据库MySQL 8.0业务数据持久化
缓存Redis 7Session、骑手实时位置缓存、订单状态缓存
实时通信Socket.IO（WebSocket）骑手 GPS 位置推送、订单状态实时更新、新订单通知
文件存储服务器本地 + Nginx 静态托管图片上传到本地 /uploads 目录，Nginx 直接提供静态文件服务

为什么选微信原生而非 uni-app？原生开发调用微信 API 更直接（支付、地图、订阅消息等），无编译层，调试体验更好，包体积更小。代价是仅支持微信小程序一端，不能编译到其他平台。

4.2 订单状态流转

pending_payment 待支付→ paid 已支付→ accepted 商户接单→ preparing 备餐中→ ready 待取餐→ delivering 配送中→ delivered 已送达→ completed 已完成

4.3 Prisma Schema 示例

// prisma/schema.prisma
model Order {
  id           Int         @id @default(autoincrement())
  userId       Int
  merchantId   Int
  riderId      Int?
  totalAmount  Decimal     @db.Decimal(10, 2)
  deliveryFee  Decimal     @db.Decimal(10, 2)
  status       OrderStatus @default(pending_payment)
  address      String
  addressDetail String
  remark       String?
  createdAt    DateTime    @default(now())
  updatedAt    DateTime    @updatedAt

  user         User        @relation(fields: [userId], references: [id])
  merchant     Merchant    @relation(fields: [merchantId], references: [id])
  rider        Rider?      @relation(fields: [riderId], references: [id])
  items        OrderItem[]
  payment      Payment?
}

enum OrderStatus {
  pending_payment
  paid
  accepted
  preparing
  ready
  delivering
  delivered
  completed
  cancelled
  refunded
}

5.1 动态 Tab 实现（关键难点）

微信小程序原生 tabBar 不支持运行时动态增减。解决方案：在 app.json 中声明 "custom": true，使用自定义 tabBar 组件，根据用户角色动态渲染不同的 tab 项目。

// app.json
{
  "tabBar": {
    "custom": true,
    "list": [
      { "pagePath": "pages/index/index" },
      { "pagePath": "pages/order/list/index" },
      { "pagePath": "pages/merchant/dashboard/index" },
      { "pagePath": "pages/rider/dashboard/index" },
      { "pagePath": "pages/profile/index" }
    ]
  }
}

// custom-tab-bar/index.js
Component({
  data: { tabs: [], selected: 0 },
  attached() {
    const { role } = getApp().globalData.userInfo || {};
    const base = [
      { text: '首页', icon: 'home', path: 'pages/index/index' },
      { text: '订单', icon: 'list', path: 'pages/order/list/index' },
      { text: '我的', icon: 'user', path: 'pages/profile/index' },
    ];
    if (role === 'merchant' || role === 'both') {
      base.splice(2, 0, {
        text: '商户', icon: 'shop', path: 'pages/merchant/dashboard/index'
      });
    }
    if (role === 'rider' || role === 'both') {
      base.splice(-1, 0, {
        text: '接单', icon: 'bike', path: 'pages/rider/dashboard/index'
      });
    }
    this.setData({ tabs: base });
  },
  methods: {
    switchTab(e) {
      const path = e.currentTarget.dataset.path;
      wx.switchTab({ url: '/' + path });
    }
  }
})

六、微信支付与分账

6.1 支付流程

① 前端提交订单→ ② 后端 JSAPI 统一下单→ ③ 返回签名支付参数→ ④ wx.requestPayment→ ⑤ 微信异步回调 Webhook→ ⑥ 验签 + 更新订单状态 
// payment.service.ts（后端 - 统一下单）
async createPrepay(orderId: number, openid: string) {
  const order = await this.prisma.order.findUnique({ where: { id: orderId } });
  const result = await this.wxpay.transactions_jsapi({
    appid: process.env.WX_APPID,
    mchid: process.env.WX_MCHID,
    description: `订单 #${orderId}`,
    out_trade_no: `ORDER_${orderId}_${Date.now()}`,
    notify_url: `${process.env.SERVER_URL}/payment/callback`,
    amount: { total: Math.round(Number(order.totalAmount) * 100), currency: 'CNY' },
    payer: { openid },
  });
  return this.wxpay.buildJsapiParams(result.prepay_id); // 返回前端支付参数
}

// 支付回调（Webhook）
async handleCallback(body: any, headers: any) {
  const verified = await this.wxpay.verifySign(headers, body); // 验签
  if (!verified) throw new BadRequestException('签名验证失败');
  const { out_trade_no, transaction_id } = body.resource.decryptedData;
  const orderId = parseInt(out_trade_no.split('_')[1]);
  await this.prisma.order.update({
    where: { id: orderId },
    data: { status: 'paid' }
  });
  await this.prisma.payment.update({
    where: { orderId },
    data: { wxTransactionId: transaction_id, status: 'paid', paidAt: new Date() }
  });
}

6.2 分账资金流向

以用户支付 ¥35（商品 ¥30 + 运费 ¥5），设商品抽成 5%、运费抽成 10% 为例：

接收方计算方式到账金额
商户¥30 × (1 - 5%) = ¥30 × 0.95¥28.50
骑手¥5 × (1 - 10%) = ¥5 × 0.90¥4.50
平台¥30×5% + ¥5×10% = ¥1.50 + ¥0.50¥2.00

6.3 关键 API 接口

接口用途
POST /v3/pay/transactions/jsapi统一下单，获取 prepay_id
GET /v3/pay/transactions/id/{id}主动查询支付结果
POST /v3/profitsharing/orders申请分账（订单完成后触发）
GET /v3/profitsharing/orders/{no}查询分账结果
POST /v3/profitsharing/return-orders分账回退（退款时调用）
POST /v3/refund/domestic/refunds申请退款
分账功能需在微信支付商户后台单独申请权限，审核周期约 5-10 个工作日，请提前准备。商户和骑手需绑定分账接收方账号后才能收到资金。建议订单完成 T+1 后触发分账，留出退款缓冲期。

七、实时位置功能

7.1 技术架构

角色行为
骑手端（小程序）接单后启动 setInterval（5秒），调用 wx.getLocation(type:'gcj02') 获取坐标，通过 Socket emit('rider_location', {orderId, lat, lng})
服务端（NestJS Gateway）接收骑手坐标 → 写入 Redis（key: rider:{id}:loc，TTL 30s）→ 推送到 Socket 房间 room_order_{orderId}
用户端（小程序）进入追踪页后 socket.emit('join_order', orderId) 加入房间，监听 location_update 事件更新 map 组件 markers

7.2 核心代码实现

// ===== 骑手端：位置上报 =====
Page({
  data: { _trackTimer: null },

  startTracking(orderId) {
    this.data._trackTimer = setInterval(() => {
      wx.getLocation({
        type: 'gcj02',
        success: ({ latitude, longitude }) => {
          getApp().socket.emit('rider_location', {
            orderId, lat: latitude, lng: longitude
          });
        }
      });
    }, 5000);
  },

  stopTracking() {
    clearInterval(this.data._trackTimer);
    this.data._trackTimer = null;
  },

  onHide() { this.stopTracking(); },
  onUnload() { this.stopTracking(); }
});

// ===== 用户端：地图追踪 =====
Page({
  onLoad({ orderId }) {
    this.orderId = orderId;
    const socket = getApp().socket;
    socket.emit('join_order', orderId);
    socket.on('location_update', ({ lat, lng }) => {
      this.setData({
        riderMarker: [{ id: 1, latitude: lat, longitude: lng,
          iconPath: '/images/rider.png', width: 32, height: 32,
          callout: { content: '骑手位置', display: 'ALWAYS', fontSize: 12 }
        }]
      });
    });
  },
  onUnload() {
    getApp().socket.emit('leave_order', this.orderId);
    getApp().socket.off('location_update');
  }
});
// ===== NestJS Socket.IO Gateway =====
@WebSocketGateway({ cors: true })
export class LocationGateway {
  @WebSocketServer() server: Server;

  @SubscribeMessage('rider_location')
  async handleLocation(client: Socket, data: { orderId: number, lat: number, lng: number }) {
    const riderId = client.data.userId;
    // 缓存到 Redis（30秒过期，骑手掉线自动清除）
    await this.redis.setex(`rider:${riderId}:loc`, 30,
      JSON.stringify({ lat: data.lat, lng: data.lng })
    );
    // 推送到对应订单的用户
    this.server.to(`room_order_${data.orderId}`)
      .emit('location_update', { lat: data.lat, lng: data.lng });
  }

  @SubscribeMessage('join_order')
  handleJoin(client: Socket, orderId: number) {
    client.join(`room_order_${orderId}`);
  }
}
骑手离线、订单完成或取消后，务必 clearInterval 停止位置上报，并调用 socket.emit('leave_order') 离开房间，避免 GPS 持续消耗电量和产生无效 Redis 写入。

// upload.controller.ts
@Post('upload')
@UseGuards(JwtAuthGuard)
@UseInterceptors(FileInterceptor('file', {
  storage: diskStorage({
    destination: (req, file, cb) => {
      const type = (req.query.type as string) || 'misc'; // products/licenses/id-cards
      const month = new Date().toISOString().slice(0, 7);  // 2025-06
      const dir = path.join(process.cwd(), 'uploads', type, month);
      fs.mkdirSync(dir, { recursive: true });
      cb(null, dir);
    },
    filename: (req, file, cb) => {
      const ext = path.extname(file.originalname).toLowerCase();
      cb(null, `${uuidv4()}${ext}`);  // 随机文件名，防止路径遍历攻击
    },
  }),
  fileFilter: (req, file, cb) => {
    const allowed = ['image/jpeg', 'image/png', 'image/webp'];
    cb(null, allowed.includes(file.mimetype));
  },
  limits: { fileSize: 10 * 1024 * 1024 },  // 最大 10MB
}))
uploadFile(@UploadedFile() file: Express.Multer.File) {
  // 返回可访问的 URL（由 Nginx 静态托管）
  const relativePath = file.path.replace(process.cwd() + '/uploads/', '');
  return { url: `${process.env.BASE_URL}/uploads/${relativePath}` };
}

9.2 NestJS 权限实现

// roles.guard.ts
@Injectable()
export class RolesGuard implements CanActivate {
  constructor(private reflector: Reflector) {}
  canActivate(context: ExecutionContext): boolean {
    const requiredRoles = this.reflector.get<string[]>('roles', context.getHandler());
    if (!requiredRoles) return true;
    const { user } = context.switchToHttp().getRequest();
    return requiredRoles.some(role => user.role === role);
  }
}

9.3 安全注意事项

场景措施
接口鉴权全局 JwtAuthGuard，每个接口 + @SetMetadata('roles', [...]) 声明所需角色
支付回调安全使用微信平台公钥对 Webhook 请求进行 RSA 验签，防止伪造回调攻击
幂等处理支付回调、分账操作做幂等校验，同一订单的回调重复触发不会重复处理
文件上传MIME 类型白名单校验 + 文件大小限制（10MB）+ UUID 随机文件名防路径遍历
敏感数据身份证号、手机号在数据库中 AES-256 加密存储，日志中脱敏
接口限流@nestjs/throttler 全局限流（100次/分钟），下单/支付接口单独加严（10次/分钟）
位置隐私Socket 房间按 orderId 隔离，用户只能看到为自己配送的骑手位置
数据越权商户只能操作自己的店铺/商品/订单，接口中校验 merchantId === req.user.merchantId
HTTPS微信小程序要求所有请求 HTTPS / WSS，Nginx 配置 SSL 证书（Let's Encrypt 免费）
